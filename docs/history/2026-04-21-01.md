## 3. AWS RDS 전환

### 배경
요구사항: "시스템 장애 발생 시 데이터를 복구할 수 있는 기능"
Docker PostgreSQL = 단일 장애점 + 일별 스냅샷만 가능

### 작업
1. RDS 보안그룹 생성 (`sg-0b0b6c1f42f9f6204`): EC2 SG에서 5432 허용
2. RDS 서브넷 그룹 생성 (`sscm-rds-subnet-group`)
3. RDS 인스턴스 생성: `sscm-postgres`, db.t3.micro, PostgreSQL 16.8, 단일 AZ, 7일 백업
4. Docker PostgreSQL → RDS 데이터 마이그레이션 (pg_dump + psql)
5. docker-compose.prod.yml 수정:
   - postgres 서비스 제거
   - `SPRING_DATASOURCE_URL`: `jdbc:postgresql://${RDS_HOST}:5432/${DB_NAME:-sscm}`
   - backend `depends_on: postgres` 제거
   - `start_period: 120s`
6. .env.prod에 `RDS_HOST` 추가
7. Docker PostgreSQL 컨테이너 및 볼륨 제거

### 핵심 교훈
`docker-compose environment:` 섹션의 `${VAR}` 치환은 shell 환경에서 발생. `env_file`은 컨테이너 내부용. 배포 시 `--env-file .env.prod` 필수.
